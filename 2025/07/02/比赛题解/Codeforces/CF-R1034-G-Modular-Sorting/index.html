[CF R1034] G. Modular Sorting

Statement

给定一个数列 {an}\{a_n\}{an​} 和一个数 mmm。保证 ai<ma_i < mai​<m。进行如下两个操作：

  1. 1 i x1~i~x1 i x。令 aia_iai​ 等于 xxx；
  2. 2 k2 ~ k2 k。选定一些 aia_iai​，令 ai=ai+k(mod  m)a_i = a_i + k (\mod m)ai​=ai​+k(modm)（该操作可以对一个元素施加多次），询问是否存在一种操作序列，使得数列变为非降数列。

注意，第二个操作并不会实际上改变原数列。

2≤n≤105,2≤m≤5×105,1≤q≤1052 \leq n \leq 10^5, 2 \leq m \leq 5 \times 10^5, 1 \leq q \leq 10^52≤n≤105,2≤m≤5×105,1≤q≤105

Tutorial

首先，我们尝试确定对于每一个 kkk，ai+k⋅x(mod  m)a_i + k \cdot x(\mod m)ai​+k⋅x(modm) 可以取哪些值。由裴蜀定理，ai+k⋅x(mod  m)a_i + k \cdot x (\mod m)ai​+k⋅x(modm) 可以取到所有 ai+xgcd⁡(k,m)a_i + x\gcd(k,m)ai​+xgcd(k,m) 的值。令 d=gcd⁡(k,m)d = \gcd(k,m)d=gcd(k,m)，于是不妨将 aia_iai​ 在不影响结果的情况下变为最小值，即 aimod  da_i \mod dai​modd。于是得到了一个新的数列。

从左向右扫描这个数列，如果 ai+1<aia_{i+1} < a_iai+1​<ai​，那么 ai+1a_{i+1}ai+1​ 需要加 ddd，同时，由于在它之后的数字都小于 ddd，所以后缀的数字都需要同时加 ddd。扫描完后，ana_nan​ 显然是最大值。如果 an<ma_n < man​<m，该情况就是合法的；如果 an≥ma_n \geq man​≥m，可以轻松得到，该情况不合法，而且显然无法构造出更合法的情况了。

然而，如果对于每一个 222 操作，都重新计算一遍，那么计算复杂度就达到了 O(qn)O(qn)O(qn)，需要优化。考虑在 111 操作时维护，222 操作时 O(1)O(1)O(1) 的查询。

可以想到，ddd 一定是 mmm 的因子，而 mmm 的因子数在该题的数据范围中不会超过 200200200，所以可以先处理出 mmm 的因子。而我们维护的对象就是一个数列里面一共加了多少次 ddd，对于一次操作 111，实际上只需要改变 aia_iai​ 和 ai+1a_{i+1}ai+1​ 是否需要加 ddd 的情况。而在 222 操作中，只需要先处理出 gcd⁡(k,m)\gcd(k,m)gcd(k,m)，即 ddd，然后直接查询操作 111 时计算好的数值即可。

时间复杂度：O((n+q)σ(m))O((n + q) \sigma(m))O((n+q)σ(m))，其中，σ(m)\sigma(m)σ(m) 是 mmm 的因子数，当 1≤m≤5×1051 \leq m \leq 5 \times 10^51≤m≤5×105 时，1<σ(m)≤2001 < \sigma(m) \leq 2001<σ(m)≤200
