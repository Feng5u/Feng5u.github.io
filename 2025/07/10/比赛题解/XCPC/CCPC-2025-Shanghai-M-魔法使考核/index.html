[CCPC 2025 Shanghai] M. 魔法使考核

Statement

给定一个数列 {an}\{a_n\}{an​}，和 nnn 个 000。每次可以进行以下两种操作：

  1. 给某个数字加一，代价为 xxx；
  2. 给某个区间内的数字翻倍，代价为 yyy。

最终使得 000 都变成对应的 aia_iai​，求最小代价。

1≤n≤3×105,0≤ai≤109,0≤x,y≤1071 \leq n \leq 3 \times 10^5, 0 \leq a_i \leq 10^9, 0 \leq x,y \leq 10^71≤n≤3×105,0≤ai​≤109,0≤x,y≤107

Tutorial

反向考虑将 aia_iai​ 都化成 000。于是两个操作分别为减一和折半。

首先，折半一定是区间越长越好，因为不管区间多长，代价不变，所以最好是令所有元素都变成可以折半的（即偶数）之后进行折半。

奇数一定要首先化成偶数，只能减一。当所有数字都变成偶数之后，只有两种可能：全部折半或全部减一达到类似折半的效果。尽管对于某些元素单独操作时，减一可能优于折半，但当考虑全体时，可以简单得到只有上述两种方案可能是最优的，每次贪心选择即可。

同时，对所有元素采用以上两种操作时，只需要考虑所有元素的总和（记为 sumsumsum）即可，即如果 sum/2∗x>ysum / 2 * x > ysum/2∗x>y，则折半更优，反之做减法。最多进行 log⁡max⁡i=1nai\log \max_{i=1}^n a_ilogmaxi=1n​ai​ 次，所有元素就都归零了。每次折半之后再扫描所有元素，将奇数化为偶数。

时间复杂度：O(nlog⁡n)O(n\log n)O(nlogn)
